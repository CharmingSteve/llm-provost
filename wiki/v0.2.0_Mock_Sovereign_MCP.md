## Objective

To prove that our proxy captures Intent (LLM), Payload (API Request), Result (API Response), and Feedback (LLM Response), the MCP server *cannot* hardcode responses. It must make actual HTTP requests to a backend API, routed through our OpenResty proxy.

## Architecture
1. **Mock Sovereign API (`mock_api.py`):** A tiny FastAPI server acting as the Bank/Ledger.
2. **Mock MCP Server (`mock_mcp.py`):** Exposes 2 tools to the LLM. When called, it makes HTTP requests to the Mock API.
3. **Docker Compose:** Packages both services alongside the LLM Provost OpenResty proxy.
